# 03. 推薦ロジック: どうやっておすすめ順を決めるか

## 基本思想

`geo-line-ranker` の推薦は、AI ではありません。  
人間が理解できるルールを組み合わせて、候補をスコアリングします。

イメージは次の通りです。

```text
候補を集める
  ↓
遠すぎる候補を落とす
  ↓
地理・路線・行動・人気・鮮度で点数を付ける
  ↓
同じ学校ばかり出ないように調整する
  ↓
推薦理由を添えて返す
```

## 1. 文脈を決める

最初に「ユーザーはどの地域・路線を見ているのか」を決めます。

優先順位はおおむね次の通りです。

1. 今回の検索条件
2. 今回の駅・路線指定
3. アカウントの居住地関連情報
4. 直近の閲覧・検索履歴
5. 同一都道府県や近隣エリアの人気
6. 全国人気。ただし遠距離候補は抑制

これを **推薦文脈** と呼びます。

## 2. 候補を絞る

次に、推薦文脈に合う候補だけを集めます。

例:

- 田町駅を指定 → 田町駅に近い学校・イベント
- 山手線を指定 → 山手線沿線の学校・イベント
- 東京都を指定 → 東京都内または近隣の候補
- 何も指定がない → 人気順フォールバック

この時点で、遠すぎる候補を出しすぎないようにします。  
ランキングで頑張る前に、候補母集団を正しく作ることが重要です。

## 3. スコアを付ける

候補には複数の観点で点数を付けます。

| 観点 | 例 | 意味 |
|---|---|---|
| 地理一致 | 同じ市区町村、近距離 | 通いやすさ |
| 路線一致 | 同一路線、同駅、近接駅 | 探索条件との一致 |
| 行動一致 | 最近見た・保存した | ユーザー関心 |
| イベント鮮度 | 開催日が近い、注目イベント | 今見る価値 |
| 人気 | エリア内PV、保存数、申込数 | 一般的な関心 |
| 多様性 | 同じ学校を出しすぎない | 探索体験の保護 |

スコアは「なぜその点が入ったか」を説明できるようにします。

## 4. 配置ごとに調整する

推薦枠の場所によって、良い推薦は変わります。

| placement | 例 | 方向性 |
|---|---|---|
| `home` | トップページ | イベントや新着を少し強めてもよい |
| `search` | 検索結果 | 検索条件に合う学校を強く出す |
| `detail` | 学校詳細 | 近い学校・同路線・関連イベント |
| `mypage` | マイページ | 保存・閲覧履歴に寄せる |

この違いを **placement profile** として設定します。  
コードを変えずに設定で調整できるのが理想です。

## 5. 多様性制御をかける

スコアが高い順に並べるだけでは、同じ学校や同じグループが並びすぎることがあります。

そのため、最後に上限制御を行います。

例:

- 同じ学校から最大2件まで
- 同じ学校グループから最大3件まで
- イベントは全体の60%まで
- 学校は最低1件出す

この制御により、ユーザーに「見る選択肢」を残します。

## 6. 推薦理由を作る

推薦理由は、スコア内訳から作ります。

例:

```json
{
  "feature": "direct_station_bonus",
  "value": 3.0,
  "reason": "田町駅に直結しています。"
}
```

ユーザー表示用には、次のような文言にできます。

- 「田町駅に近いイベントです」
- 「山手線沿線の学校です」
- 「最近見た学校に近い候補です」
- 「このエリアで人気があります」

重要なのは、AIで作文しないことです。  
固定文言をルールに紐づけることで、表現が安定します。

## 7. フォールバック

データが足りないときは、推薦枠を空にしないようにします。

ただし、何でも出してよいわけではありません。  
極端に遠い候補を避けるルールを残します。

フォールバック例:

| 段階 | 内容 |
|---|---|
| strict | 指定駅・指定路線・指定エリアに強く一致 |
| relaxed | 近隣駅・近隣市区町村まで広げる |
| area_popular | 同一都道府県や近隣エリアの人気 |
| national_safe | 全国人気。ただし遠距離や不自然な候補を抑制 |

## 8. 学校とイベントは混ぜすぎない

学校とイベントは性質が違います。

- 学校は安定した情報
- イベントは鮮度が重要
- イベントは一時的に強くなりやすい
- 学校は長期的な探索軸になりやすい

そのため、学校とイベントは別々にスコアリングしたうえで、placement profile に従って混ぜます。

## 9. 変更時の注意

ランキングルールを変えるときは、必ず次を確認します。

- 遠い候補が増えていないか
- 同じ学校ばかりになっていないか
- イベントだけに偏っていないか
- 推薦理由が嘘になっていないか
- フォールバックが過剰に全国人気へ落ちていないか
- algorithm version / profile version が更新されているか

## まとめ

この推薦ロジックは、ブラックボックスではありません。  
「どの候補を集め、どの観点で点数を付け、どの上限制御で整えたか」を説明できます。

地味ですが、強いです。  
壊れにくい推薦は、派手な魔法より透明な定規から始まります。
